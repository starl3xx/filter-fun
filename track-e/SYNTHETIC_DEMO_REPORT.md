## Methodology

- HP components computed per spec §6.4 + §41.4 implementations
- Velocity / effectiveBuyers / stickyLiquidity normalized via corpus-relative percentile rank (so composite makes sense across the corpus). Other 3 components already 0-1 by construction.
- Component → outcome correlations: Spearman ρ (rank-based, robust to outliers) + ROC AUC
- Feature importance: RandomForest with 200 trees, default depth, averaged across horizons
- Weight fitting: L2-regularized logistic regression on standardized components, coefficients clipped to non-negative, normalized to sum to 1.0
- Rank stability: Spearman ρ between HP rankings under two different weight sets
- Cross-validation: 5-fold stratified, mean ± std AUC
